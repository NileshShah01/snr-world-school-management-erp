18 UI/UX Problems Found

Ranked by user impact. Critical = users can't do their job. High = friction every session. Medium = polish gap.

1
Sidebar has no visual hierarchy — 17 categories look identical Critical

All 17 sidebar categories use the same font weight, size, and color. Users can't scan to find what they need. The competitor EducationDesk groups items into logical visual clusters with color-coded icons per module.

🔴 Your CurrentFlat list of 17 categories, all looking identical. No grouping, no color coding, no visual weight difference between parent and child items.
✅ What's NeededColor-coded module icons (blue=students, green=fees, orange=exams), subtle section dividers, and clear open/closed state for each category.
2
No breadcrumb — users don't know where they are Critical

When an admin is on "Bulk Result Generator" there's zero indication of their location: no breadcrumb, no page title, no "you are here" signal. After clicking through the sidebar, context is completely lost.

🔴 CurrentMain content area shows just the section content. No "Home → Results → Bulk Generator" trail.
✅ FixAdd Home / Results / Bulk Generator breadcrumb in the page header. 3 lines of JS to wire it to showSection().
3
Dashboard home shows no actionable data Critical

The admin's first screen after login is mostly empty space or generic stats. The competitor opens with: today's attendance %, fees collected this month vs. target, overdue count, and upcoming exam alerts. Yours doesn't do any of this.

🔴 CurrentStatic stat cards with placeholder numbers. No "urgent actions needed" panel, no today's summary, no activity feed.
✅ Fix4 live KPI cards (attendance today, fees this month, overdue, exams this week) + 1 quick-action row + recent-activity feed. All Firestore-queryable.
4
Forms have no visual structure — 28-field add-student is a wall of inputs Critical

The Add Student form dumps all 28 fields on one screen in a single column with no sections, no steps, no grouping. Users make errors and miss required fields. Competitor uses a multi-step wizard with logical groups: Basic Info → Academic Info → Guardian Info → Photo/Docs.

🔴 Current28 fields in one form, linear layout, no visual separation between personal, academic and guardian data.
✅ Fix4-step wizard with a step progress indicator: Step 1 Basic, Step 2 Academic, Step 3 Guardian, Step 4 Documents. Each step = 5-8 fields max.
5
No global search across the app High

To find a student, admin must navigate: Sidebar → Student Management → Search Student → type name. There's no ⌘K / Ctrl+K command palette or top-bar search. Every competitor has a global search bar pinned to the top of the dashboard.

🔴 CurrentPer-section search only. To find a student you must already be in the Student section.
✅ FixGlobal search bar in the top header. Searches students, fees, and sections simultaneously. Keyboard shortcut Ctrl+K.
6
Data tables have no sorting, filtering, or export buttons High

Student lists, fee tables, attendance grids — all are rendered as basic HTML tables with no column sorting, filter chips, pagination display, or "Export to CSV/Excel" button visible in the table header. Competitor uses DataTables.js with per-column filters and Excel/PDF export in every table header.

🔴 CurrentPlain table, no sort arrows, no visible pagination, no export button, no column filters.
✅ FixTable header with sort-by arrows + filter chips + "Export CSV" and "Print" buttons. Row count display: "Showing 1-25 of 347 students".
7
Fee payment screen is confusing — 3 separate sections for one task High

Collecting a fee payment requires navigating: 1) Search Student, 2) Class Fee Payment, 3) Demand Receipt — three separate sidebar items for what should be a single 3-step flow. The competitor does it in one page: search → ledger → collect → receipt, all in-page.

🔴 CurrentFee flow is split across 3+ separate sections in sidebar. Staff must jump around to complete one payment transaction.
✅ FixSingle "Collect Fee" page: Step 1 find student (type-ahead), Step 2 see outstanding (auto-loaded), Step 3 enter amount + mode, Step 4 print receipt. All in one screen.
8
Student portal tabs are invisible on mobile High

The student dashboard sidebar collapses on mobile but the tab labels are hidden, leaving only icons. Many icons are ambiguous — a folder icon for "Resources" isn't intuitive. Students can't tell where they are without labels.

🔴 CurrentSidebar collapses to icon-only on mobile. No bottom nav bar. No tooltips on icons. Labels disappear entirely.
✅ FixSwitch to a bottom navigation bar on mobile (max 5 items: Dashboard, Fees, Exams, Attendance, More). Bottom nav is the mobile-native pattern.
9
Empty states show nothing — just blank content areas High

When a section has no data (e.g. no exams created yet, no library books added), the content area is blank or shows a tiny "No records" text. Users don't know if the section is loading, broken, or genuinely empty. There's no call-to-action to add the first item.

🔴 Current"No data" in grey text, no icon, no context, no action button. Users feel lost.
✅ FixIllustrated empty states: icon + heading "No exams yet" + description + primary action button "Create First Exam →"
10
Loading states are generic — same spinner for all operations High

A full-page overlay spinner appears for every operation from "loading dashboard" to "saving one field". The 12-second fail-safe is too long. Users can't tell if something is working or frozen.

🔴 CurrentFull-page white overlay + spinner for ALL operations. Same experience for 0.2s Firestore read as for 5s PDF generation.
✅ Fix3 tiers: (1) inline skeleton loader for data tables, (2) button spinner for form submits, (3) full-page overlay only for navigation/PDF generation.
11
Typography is inconsistent — 14 different font sizes in portal.css High

portal.css has font sizes: 0.55rem, 0.6rem, 0.65rem, 0.7rem, 0.75rem, 0.8rem, 0.85rem, 0.875rem, 0.9rem, 0.95rem, 1rem, 1.1rem, 1.15rem, 1.25rem, 1.5rem, 1.8rem — 16 values with no scale. Text looks unpredictably sized across sections.

🔴 Current16 arbitrary font sizes with comments like "Reduced from 0.875rem". No type scale, no design tokens.
✅ FixDefine 6 sizes as CSS vars: --text-xs(0.72rem), --text-sm(0.83rem), --text-base(0.95rem), --text-md(1.1rem), --text-lg(1.3rem), --text-xl(1.6rem). Use only these 6.
12
Fee table has no "Paid / Due / Partial" status column with color High

Fee records are shown in plain rows with no status indicators. Staff must read the numbers to determine if a student is paid up or overdue. The competitor shows a colored status badge (green = Paid, red = Overdue, yellow = Partial) in every row.

🔴 CurrentRaw number columns. Staff visually scan amounts to determine status.
✅ FixAdd status pill column: ● Paid ● Overdue ● Partial
13
Buttons lack consistent hierarchy — primary/secondary/ghost all look similar Medium

Forms have multiple buttons (Save, Cancel, Export, Delete) that look nearly identical in size and color. There's no clear visual hierarchy telling users which is the primary action. Delete buttons aren't styled destructively.

🔴 CurrentSave, Cancel, Export, Delete — all similar-looking buttons side by side.
✅ FixPrimary (blue, filled), Secondary (blue, outlined), Ghost (no border), Destructive (red). Delete buttons must be red. Max 1 primary action per form.
14
No page title in browser tab — every tab says "Admin Dashboard" Medium

Every admin dashboard section shares the same browser title "Admin Dashboard". If an admin has 3 tabs open (student search, fee collection, result entry), all tabs say the same thing. Multi-tasking is broken.

🔴 Currentdocument.title is set once and never updated when showSection() is called.
✅ FixIn showSection(), update document.title: "Students | Apex ERP", "Fees | Apex ERP". 2 lines of JS.
15
Public site homepage stats are hardcoded, look fake Medium

The homepage counter shows "300 Students, 15 Teachers, 12 Classes, 8 Years" as hardcoded HTML values. These numbers don't update from the CMS and read as placeholders to any savvy visitor. The CMS exists to drive this, but it's not wired up.

🔴 Currentdata-target="300" hardcoded in HTML. Admin cannot change from CMS.
✅ FixWire counter to Firestore globalStats. Already managed in CMS → Global Stats section — just needs the homepage to read it. 5 lines of JS.
16
No keyboard shortcuts — power users are slowed down Medium

A school accountant collecting 50+ fees per day has no keyboard shortcuts. Every student lookup requires mouse navigation to the sidebar → search box → type. Ctrl+K for global search and arrow keys to navigate results would save 30+ seconds per transaction.

🔴 Current100% mouse-driven UI. No keyboard navigation beyond browser defaults.
✅ FixCtrl+K = global search. Ctrl+S = save current form. Escape = close modal. Arrow keys in search results. These are 20-line JS additions.
17
Toast notifications disappear in 3 seconds — too fast for long messages Medium

"Fee payment recorded for Aarav Singh — ₹3,500 received for March 2026, Receipt #1042" disappears in 3 seconds. For success confirmations with details, users can't read them in time.

🔴 CurrentFixed 3-second dismiss for all toasts regardless of message length.
✅ FixCalculate duration from message length (min 3s, +1s per 20 chars). Add a ✕ close button on every toast. Persist error toasts until dismissed.
18
Mobile public site: floating inquiry button overlaps footer content Medium

The fixed floating "Inquiry" button sits at bottom-right and overlaps footer links on mobile screens. Users can't click the footer phone/email links because the button is on top of them.

🔴 CurrentFloating button: position: fixed; bottom: 20px; right: 20px — covers footer on all screen sizes.
✅ FixHide floating button when footer is in viewport (IntersectionObserver on footer). Or push button above footer with bottom: 80px on mobile.

SNR ERP vs EducationDesk — Head to Head

Your competitor runs on Bootstrap 3 + jQuery from 2018. You have a modern stack. You should be winning every UX category — but you're currently losing most of them because functionality isn't enough.

Key takeaway: You have a fundamentally superior technical foundation (modern stack, SaaS architecture, real integrations, better security). But EducationDesk wins on day-to-day usability: searchable dropdowns, data table tools, streamlined fee collection, and breadcrumbs. These are all achievable fixes, not rewrites.

After — User always knows where they are

// In admin-dashboard.js — add to showSection() const SECTION_LABELS = { 'bulkResultGenerator': { label: 'Bulk Result Generator', parent: 'Results' }, 'classFeePayment': { label: 'Collect Fee', parent: 'Fees' }, // ... all sections }; function showSection(id) { // ... existing hide/show logic ... const meta = SECTION_LABELS[id]; if (meta) { document.title = `${meta.label} | ${schoolName} ERP`; document.getElementById('breadcrumbCurrent').textContent = meta.label; document.getElementById('breadcrumbParent').textContent = meta.parent; } }

3-Week Implementation Roadmap

Ordered by impact-to-effort ratio. Do Week 1 first — it solves the most felt pain with the least code.

  • Breadcrumb + document.title update
  • Status pills in fee/student tables
  • Empty state components (copy code above)
  • Toast: add close button + auto-duration
  • Fix floating button overlapping footer
  • Wire homepage counters to Firestore globalStats
  • Mobile bottom nav for student portal
🟡 Week 2 — Sidebar + Search Overhaul
  • Color-code sidebar module groups
  • Add count badges to sidebar items (fee dues, absent today)
  • Global search bar in admin header (Ctrl+K)
  • Add filter chips to student list
  • Export CSV button to all data tables
  • Sort arrows on table columns
  • Skeleton loaders for data tables
🟢 Week 3 — Dashboard + Forms
  • Live KPI cards on admin home (attendance, fees, exams)
  • Quick-action row on dashboard
  • Add Student form: break into 4-step wizard
  • Define type scale CSS vars (6 sizes only)
  • Consolidate fee collection into one page
  • Button hierarchy system (primary/secondary/ghost/destructive)
  • Keyboard shortcuts (Ctrl+K, Ctrl+S, Escape)

  • Select2 searchable dropdowns on all class/section fields
  • Tiered loading states (inline vs. full-page)
  • Page transition animations (subtle)
  • Hover states on table rows
  • Confirm dialogs for destructive actions
  • Accessibility audit (WCAG 2.1 AA)
  • Dark mode (optional)
